# PRD — Lexica Spire

**Версия:** 1.0
**Дата:** 2026-02-13
**Статус:** Pre-Development (MVP Phase 1)

---

## Что строим

**Lexica Spire** — образовательный roguelike deck-builder, в котором изучение английского языка является основной боевой механикой. Игрок проходит процедурно-генерируемые акты, где активация каждой карты требует решения лингвистического задания — от грамматических головоломок до словарных тестов.

**Тэглайн:** *"Master Language. Conquer the Spire."*

---

## Для кого

| Аудитория | Описание | Приоритет |
|-----------|----------|-----------|
| **Основная** | Изучающие английский (B1-C1 CEFR), 16-35 лет | Высокий |
| **Вторичная** | Геймеры, интересующиеся образовательным контентом | Средний |
| **Третичная** | Преподаватели языка, ищущие вовлекающие инструменты | Низкий |

## Ключевые фичи MVP (Phase 1)

### Боевая система "Syntax Strike"
- 3 Energy за ход, карты требуют решения лингвистического задания
- Типы карт: Attack (урон), Skill (блок/утилиты), Power (постоянные бафы)
- Система качества ответа: Perfect / Correct / Slow / Mistake
- Механика Fatigue (штраф за медленные ответы)

### Контент
- **20 карт:** 8 Common Attack, 6 Common Skill, 4 Uncommon Attack, 2 Uncommon Skill
- **6 типов врагов:** Шёпот, Утгалу, Одержимый раб, Одержимый стражник, Одержимый жрец, Разорванный (элитный)
- **1 босс Акта 1:** Ашипу (250 HP, 3 фазы, Смена Времён)
- **5 реликвий:** Golden Dictionary, Phonetic Ankh, Etymology Lens, Speed Reader's Monocle, Polyglot's Amulet
- **2 проклятия:** Echo of a Typo, Tense Fog

### Прогрессия
- Карта Акта 1 с 15 нодами (Combat, Elite, Library, Rest Site, Merchant, Event, Boss)
- Система Spaced Repetition (модифицированный SM-2)
- Адаптивная сложность на основе точности ответов

### Магазин
- Покупка/удаление карт
- Удаление проклятий (50-175 Lex-Coins по интенсивности)
- Базовый ассортимент: 5 карт + сервисы

### Сохранения
- Локальные сейвы (JSON)
- Save/Load прогресса рана

### Типы заданий (только текст)
- Vocabulary: выбор определения (multiple choice)
- Grammar: исправление предложения (текстовый ввод)
- Vocabulary matching: подбор синонимов
- Tense conjugation: спряжение глаголов

| Фича | Причина исключения | Фаза |
|------|--------------------|------|
| Speaking/Listening челленджи | Требуют Vosk/Whisper, усложняют MVP | Phase 3 |

| Облачные сейвы | Не нужны без авторизации | Phase 4 |
| Акт 2 и 3 | Контент расширяется после валидации core loop | Phase 2-3 |
| Мета-прогрессия (Insight Points, Library of Mastery) | Между ранами, после основного контента | Phase 2 |
| Альтернативные стартовые колоды | Разблокируются через мета-прогрессию | Phase 2 |
| Система достижений | Не критично для core loop | Phase 2 |
| Мультиязычность (выбор родного + изучаемого языка) | MVP = русский → английский; TranslationServer + языковые пары контента потом | Phase 4 |
| DLC-пакеты (тематические наборы слов/правил) | Business English, IELTS, Medical и т.д. — загружаемые JSON-пакеты | Phase 5 / Post-Release |
| Steam/Itch.io интеграция | Только локальный деплой на MVP | Phase 4 |
| Voice acting / TTS | Не нужны без аудио-заданий | Phase 3 |
| Мультиплеер/лидерборды | Не в скоупе | Не планируется |

## Визуальная стилистика

Стиль арта — **тёмный комикс / графическая новелла** с элементами хоррора:

| Приём | Описание |
|-------|----------|
| **Жирный контур (Inking)** | Все объекты и персонажи имеют чёткие, нарочито грубые чёрные границы |
| **Штриховка и кросс-хэтчинг** | Вместо мягких градиентов — ручная штриховка (множество мелких линий) для передачи теней и объёма |
| **Сел-шейдинг (Cel-shading)** | Цвета плотными слоями с резкими переходами между светом и тенью, как в рисованной мультипликации и комиксах |
| **Мрачная десатурированная палитра** | Грязные, приглушённые тона (оливковый, серый, коричневый). Яркие акцентные цвета выделяются на их фоне |
| **Гранж-текстуры** | Потёртости, царапины, грязь на поверхностях |
| **Фон ассетов** | Однородный ярко-салатовый фон без отбликов, теней и надписей (для чистого вырезания) |

## Будущее: Мультиязычность и DLC-контент (Phase 4+)

### Концепция

Lexica Spire в текущем MVP привязан к паре **русский → английский**. В будущих фазах система должна поддерживать **произвольные языковые пары**: игрок указывает свой родной язык (`native_language`) и изучаемый (`target_language`). Например:

- Родной: 中文 → интерфейс на китайском, задания формулируются на китайском
- Изучаемый: English → задания проверяют знание английского
- Или: Español → Deutsch, 日本語 → English, и т.д.

Дополнительно, **тематические DLC-пакеты** позволят загружать расширенные наборы слов и правил: бизнес-английский, медицинская терминология, подготовка к IELTS, сленг и т.д.

### Архитектурная модель

#### 1. Настройки языковой пары

```
user://settings.json
{
  "native_language": "zh",      // родной язык (ISO 639-1)
  "target_language": "en",      // изучаемый язык
  "ui_locale": "zh",            // язык интерфейса (= native по умолчанию)
  "cefr_level": "B1"
}
```

Игрок выбирает языки при первом запуске (или в Настройках). `ui_locale` определяет язык всех кнопок, меню, подсказок. `target_language` определяет, на каком языке будут задания.

#### 2. Локализация UI — Godot TranslationServer

Все строки интерфейса выносятся в файлы переводов через встроенный `TranslationServer` Godot:

```
assets/
  translations/
    ui.ru.translation    # русский (базовый)
    ui.en.translation    # английский
    ui.zh.translation    # китайский
    ui.es.translation    # испанский
    ui.ja.translation    # японский
    ...
```

Все строки в коде заменяются на вызовы `tr()`:
```gdscript
# Было:
button.text = "НОВАЯ ИГРА"
# Стало:
button.text = tr("MENU_NEW_GAME")  # → "新游戏" / "NEW GAME" / "NUEVA PARTIDA"
```

Файл переводов добавляется в `Project → Localization → Translations`. При смене `ui_locale` вызывается `TranslationServer.set_locale("zh")` — весь UI мгновенно переключается.

#### 3. Контент заданий — языковые пары

Задания организуются по парам `{native}_{target}`:

```
data/
  content/
    ru_en/                         # русский → английский (MVP)
      challenge_pool.json          # 120+ заданий
      cards_localized.json         # названия/описания карт
      events_localized.json        # тексты событий
    zh_en/                         # китайский → английский
      challenge_pool.json
      cards_localized.json
      events_localized.json
    es_de/                         # испанский → немецкий
      ...
```

Структура задания адаптируется под пару:
```json
{
  "id": "vocab_42",
  "type": "vocabulary",
  "prompt": "「abandon」是什么意思？",
  "options": ["放弃", "接受", "开始", "改变"],
  "correct": "放弃",
  "target_word": "abandon",
  "native_hint": "放弃，抛弃"
}
```

При загрузке `ApiClient` выбирает пул по текущей языковой паре:
```gdscript
var pair := "%s_%s" % [Settings.native_language, Settings.target_language]
var pool_path := "res://data/content/%s/challenge_pool.json" % pair
```

#### 4. DLC-пакеты — тематические расширения

**Формат DLC-пакета** — ZIP-архив или папка со стандартной структурой:

```
dlc/
  business_english_v1/
    manifest.json                  # метаданные пакета
    challenges/
      vocabulary.json              # 200+ бизнес-терминов
      grammar.json                 # деловая грамматика
    assets/                        # опционально
      card_skins/                  # скины карт (тематические)
      event_texts.json             # тематические события
```

**manifest.json:**
```json
{
  "id": "business_english_v1",
  "name": "Business English Pack",
  "name_localized": {
    "ru": "Бизнес-английский",
    "zh": "商务英语包",
    "es": "Inglés de negocios"
  },
  "version": "1.0.0",
  "language_pairs": ["*_en"],
  "difficulty_range": ["B1", "B2", "C1"],
  "challenge_count": 350,
  "tags": ["business", "corporate", "formal"],
  "price_tier": "paid"
}
```

- `language_pairs: ["*_en"]` — пакет для любого родного языка при изучении английского
- `language_pairs: ["zh_en"]` — только для носителей китайского
- `tags` — для фильтрации в магазине DLC

**Загрузка DLC:**
```
user://dlc/                        # скачанные пакеты
  business_english_v1/
  ielts_prep_v2/
  medical_terms_v1/
```

`DlcManager` (autoload) сканирует `user://dlc/`, читает `manifest.json`, регистрирует доступные пакеты. Игрок включает/выключает пакеты в Настройках. При включённом пакете его задания подмешиваются в основной пул.

#### 5. Бэкенд — масштабируемость

```
POST /api/generate-challenge
{
  "player_id": "...",
  "native_language": "zh",
  "target_language": "en",
  "challenge_type": "vocabulary",
  "difficulty": "B1",
  "dlc_packs": ["business_english_v1"],   // активные DLC
  "exclude_ids": ["vocab_1", "vocab_5"]   // уже отвеченные
}
```

Бэкенд:
- Выбирает пул заданий по `native_language` + `target_language`
- Подмешивает задания из указанных DLC-пакетов
- SM-2 трекинг ведётся **per language pair per player** (один игрок может учить несколько языков)
- Адаптивная сложность учитывает CEFR-уровень игрока для конкретной пары

#### 6. Дорожная карта реализации

| Этап | Что | Когда |
|------|-----|-------|
| **Phase 4a** | `TranslationServer` + `.translation` файлы для UI (EN, ZH, ES) | После MVP |
| **Phase 4b** | Структура `data/content/{pair}/`, миграция `challenge_pool.json` в `ru_en/` | После MVP |
| **Phase 4c** | Экран выбора языков (первый запуск + Настройки) | После MVP |
| **Phase 5a** | `DlcManager` autoload — сканирование, загрузка, регистрация пакетов | Alpha |
| **Phase 5b** | UI магазина DLC (витрина + включение/выключение) | Alpha |
| **Phase 5c** | Бэкенд: API `/api/dlc/catalog`, `/api/dlc/download` | Alpha |
| **Post-Release** | Первые DLC: Business English, IELTS Prep, Travel Phrases | Релиз+ |

#### 7. Принципы масштабируемости

- **Данные отделены от кода:** все задания, строки, тексты событий — в JSON/translation-файлах, не в GDScript
- **Языковая пара — ключ контента:** один и тот же геймплей, разные пулы заданий
- **DLC = данные, не код:** пакет не содержит скриптов, только JSON + ассеты
- **Fallback-цепочка:** если перевода нет, используется английский → если нет пары, предлагается создать
- **Горячая подгрузка:** DLC загружаются без перезапуска игры

## Ограничения и риски

| Риск | Митигация |
|------|-----------|
| Соло-разработка — медленный прогресс | AI-помощник (Claude), AI-арт, фокус на core loop |
| Базовый опыт с Godot | GDScript проще для входа, много туториалов |
| Баланс обучения vs. геймплея | Раннее плейтестирование, итерации |
| Качество AI-арта | Ручная доработка, единый стиль-гайд |
